## 2024-04-14 - Predictable Fallbacks in Cryptographic Entropy Sources
**Vulnerability:** Several functions (`newRelayServer` and `randomHex` in `cmd/truco-relay/main.go`, and `randomKey` in `browser-edition/cmd/httpapi/main.go`) used predictable fallback values (hardcoded strings or timestamps) if `crypto/rand` failed to generate entropy.
**Learning:** Falling back to predictable values when entropy generation fails compromises the security of cryptographic operations, session keys, and secrets. It creates a silent failure where the system appears to work but is fundamentally insecure.
**Prevention:** If an entropy source fails during cryptographic operations or secret generation, the application must panic and fail-closed rather than continuing with insecure fallback values.

## 2025-02-12 - Missing Expiration Validation with InsecureSkipVerify
**Vulnerability:** When configuring a custom TLS client with `InsecureSkipVerify: true` and a custom validation function via `VerifyPeerCertificate`, the client successfully validated the custom certificate fingerprint (e.g., for ad-hoc P2P network authorization) but implicitly bypassed native validation of the certificate's `NotBefore` and `NotAfter` expiration dates. An attacker with a previously compromised or leaked short-lived key (which normally rotates every 72 hours) could continue to authenticate indefinitely.
**Learning:** `VerifyPeerCertificate` only provides the raw certificate bytes, requiring manual parsing to inspect attributes, and does not enforce expiration on its own when the native verify logic is skipped. It is easy to overlook that `InsecureSkipVerify` skips *all* validation, not just CA roots.
**Prevention:** Always use `VerifyConnection` (available in Go 1.15+) instead of `VerifyPeerCertificate` when customizing verification logic. `VerifyConnection` provides `tls.ConnectionState` which contains fully parsed `x509.Certificate`s in `PeerCertificates`, making it trivial to explicitly implement custom expiration checks (e.g., `now.Before(cert.NotBefore) || now.After(cert.NotAfter)`).
